#### 2. NA მნიშვნელობების შევსების სტრატეგია
* **რიცხვითი სვეტები შევავსე 0-ით:** გარკვეულ რიცხვით სვეტებში, სადაც მონაცემი აკლდა, მნიშვნელობა შეივსო 0-ით. მაგ.თუ სახლს არ აქვს გარაჟი, GarageYrBlt შეივსო ნულით.
* **მედიანითა და მოდით შევსება:** იმ სვეტებისთვის, სადაც 0-ით ან "None"-ით შევსება არალოგიკური იქნებოდა, გამოვიყენეთ სტატისტიკური მაჩვენებლები. მაგ. LotFrontage (ქუჩის მხარეს ფასადის სიგრძე) შეივსო მედიანური მნიშვნელობით.
    * ტესტ-სეტის შესავსებისას გამოვიყენე სატრენინგო მონაცემების მედიანა, რათა თავიდან ამეცილებინა Data Leakage.
* **კატეგორიული ცვლადების შევავსე "None"-ით:** ამით მოდელს ვაწვდით ინფორმაციას, რომ მახასიათებლის არქონა თავისთავად მნიშვნელოვანი ფაქტორია ფასის განსაზღვრისას.

#### 3. outlier-ების მოშორება
სხვადასხვა feature-ზე ვიზუალიზაციის (Scatter Plots) მეშვეობით დავაფიქსირე ისეთი სახლები, რომლებსაც ჰქონდათ ძალიან დიდი ფართობი, თუმცა მათი ფასი არაპროპორციულად დაბალი იყო. ასეთი მონაცემები მოდელს აძლევს მცდარ სიგნალს და იწვევს Overfitting-ს ამ ტიპის ექსტრემალურ წერტილებზე.

#### 4. კატეგორიული ცვლადების რიცხვითში გადაყვანა
* Label Encoding: გამოვიყენე იმ სვეტებისთვის, რომლებსაც გააჩნიათ გარკვეული თანმიმდევრობა. მაგალითად: ხარისხის მაჩვენებლები (Ex, Gd, TA, Fa, Po), სადაც "Excellent" უფრო მაღალია, ვიდრე "Good". ამით მოდელმა ისწავლა კატეგორიებს შორის არსებული იერარქიული კავშირი.

* One-Hot Encoding: გამოვიყენე დანარჩენი კატეგორიული სვეტებისთვის (მაგ: Neighborhood, Foundation, HouseStyle), სადაც კატეგორიებს შორის არანაირი იერარქია არ არსებობს. თითოეული მნიშვნელობა გარდაიქმნა ცალკეულ სვეტად, რაც მოდელს საშუალებას აძლევს, თითოეული მახასიათებელი დამოუკიდებლად აღიქვას.


#### 5. ლოგარითმული ტრანსფორმაცია (Log Transformation)
სამიზნე ცვლადი: რადგან სახლის ფასების განაწილება იყო ასიმეტრიული, გამოვიყენე np.log1p() ტრანსფორმაცია. ეს ეხმარება მოდელს, რომ მცირე და დიდი ფასების ცდომილება დრამატულად აღარ აღიქვას.

### ახალი Feature-ების დამატება:
მოდელის პროგნოზირების უნარის გასაუმჯობესებლად, არსებული მონაცემების საფუძველზე შევქმენი ახალი, აგრეგირებული ცვლადები:
* **TotalSF (Total Square Footage)**: დავაჯამე სარდაფის (TotalBsmtSF), პირველი (1stFlrSF) და მეორე სართულის (2ndFlrSF) ფართობები. ეს სვეტი აერთიანებს სახლის მთლიან საცხოვრებელ სივრცეს ერთ სვეტში, რაც ვიფიქრე, რომ უკეთესი მახასიათებელი იქნებოდა.

* **TotalBath (Total Bathrooms)** :ანალოგიურად, შევქმენი სველი წერტილების ერთიანი მაჩვენებელი. ფორმულაში გათვალისწინებულია როგორც სრული (FullBath), ისე ნახევარი სააბაზანოები (0.5 * HalfBath), როგორც სარდაფში, ისე ძირითად სართულებზე.

* **HouseAge (Property Age)**: გამოვთვალე სახლის ასაკი გაყიდვის მომენტისთვის (YrSold - YearBuilt). ეს ცვლადი უფრო ზუსტად ასახავს სახლის "სიძველეს", ვიდრე უბრალოდ აშენების წელი.

* **RemodAge (Years Since Remodel)**: ანალოგიურად, გამოვთვალე დროის ინტერვალი ბოლო რემონტიდან გაყიდვამდე (YrSold - YearRemodAdd). ეს მოდელს ეხმარება გაარჩიოს ძველი, მაგრამ ახლად გარემონტებული სახლები უბრალოდ ძველი და მიტოვებული სახლებისგან.

მონაცემთა შეზღუდვა inference-ის დროს. (Capping/Clipping)
იმისათვის, რომ მოდელი არ დაბნეულიყო ტესტ-სეტში არსებულ დიდ მნიშვნელობებზე (რომლებიც ტრენინგისას ამოვჭერით), გამოვიყენეთ .clip() მეთოდი.

ტესტ-მონაცემებში ყველა მნიშვნელობა, რომელიც აღემატებოდა დადგენილ ზღვარს, გაუტოლდა ამავე ზღვარს (ზედა ზღვრის ფიქსაცია).
